# 无尽雪夜（Endless Snow Night）— 游戏设计文档

## 一、世界观

一座与世隔绝的庄园，被暴风雪围困。十二个陌生人被困于此。
每七天是一个轮回。第七天结束时，一切重置——除了陈默的记忆。

### 核心循环
- **Day 1**：众人抵达庄园，彼此介绍，安顿。
- **Day 2**：陈默觉醒记忆，发现轮回。
- **Day 3-6**：白天探索+社交+推理，夜晚狼人行动。剧情随线索推进。
- **Day 7**：结局触发。真结局、好结局、坏结局、隐藏结局。
- **重置**：除了陈默，无人记得上一轮的事。

只有陈默收集到足够破局线索时，才能触发真结局，打破轮回。

## 三、核心规则

### 信息边界（最重要）
- **所有角色只知道**：自己的身份和技能、白天发生的事、线索的原始内容
- **狼队第一夜碰面**：知道彼此是狼，但不确定具体狼种
- **狼不知道神职身份和技能**，神职之间也不知道彼此身份和技能
- **所有人都不知道**：其他人的身份和技能（除非公开或被查验）

### 技能使用规则
- 神职可以任意时间发动能力（力量来源于村民的信仰）
- 狼只能夜晚发动能力（力量来源于夜光）
- 狼王白天被杀可以同归于尽
- 机械狼白天也可以使用偷来的能力

### 技能一览

**神职：**
- 预言家（方衡·回响）：探知对方是盟友还是敌人。村民怀有杀意时探测为敌人。
- 女巫（叶知秋·还魂）：感知生命流逝；接触物品下毒；接触尸体复活（复活不暴露身份）；可制作麻痹/安眠药。合计救+毒不超过3次，否则精神崩溃。
- 骑士（林小满·守护）：私下接触时可发动"决斗"斩杀狼人（对村民发动则自己死亡）。可选"守卫"保护一人过夜（不能守自己，虚弱2天）。
- 魔术师（沈慎·致幻）：每晚选择两人交换身份，使狼人袭击时幻觉误杀同伴。上一晚参与的人不可重复。

**狼：**
- 狼王（周阳）：被村民或神杀死时可同归于尽。被下毒不能发动。
- 隐狼（唐小棠）：预言家探测为"盟友"，无法被查验识破。
- 清道夫狼/狼（赵明城）：击杀后可以抹除尸体，女巫无法复活。
- 机械狼（顾言）：弑神后获得该神职能力（变体：预言家直接查看身份、女巫只能毒1次、骑士决斗逻辑相反、魔术师交换身份）。偷来的技能一次性。

## 四、AI模拟平衡性

基于模拟迭代，当前参数：
- 狼队投票：2票多数通过，平票随机目标
- 女巫：首夜必救，其后30%触发诅咒（专毒狼）
- 骑士：20%决斗，30%守卫
- 魔术师：25%交换，80%触发

## 五、结局系统

| 结局ID | 名称 | 触发条件 |
|--------|------|---------|
| true | 真结局 | 收集所有破局线索（cellar_door + altar_inscription + god_kill_method），打破轮回 |
| good | 好结局 | 利用祭坛之力逃出山庄 |
| bad | 饿死/绝望 | 第7天物资耗尽，全员死亡 |
| hidden | 弑神/万神之神 | SAN < 20 + 知道弑神方法，选择接受或拒绝成为新神 |

## 六、技术架构

### 目录结构

```
/src
  /passages
    /chapter01/       # 第一章（Day 1）
    /chapter02/       # 第二章（Day 2，循环觉醒）
    /chapter03_06/    # 第三至六章（Day 3-6，推理调查）
    /chapter07/       # 结局
    /common/          # 公共UI（状态栏、地图、道具栏、神职面板）
  /scripts
    state.js          # 状态结构定义与常量
    game.js           # 公共API（道具/线索/Flag/轮回/安全屋）
    /skills/          # 神职AI技能模块
    /wolves/          # 狼队AI模块
    pursuit.js        # 夜间追击战
  /styles
    story.css         # 游戏样式
/tools
  build.js            # 构建脚本（Twee → HTML）
  check-links.js      # 链接检查
  sim_v*.js           # AI模拟脚本
/dist
  index.html          # 可发布HTML
/docs
  design-doc.md       # 本文档
  writing-guide.md    # 写作指南
  variable-guide.md   # 变量指南
  passage-guidelines.md # Passage命名规范
```

### 运行方式
```bash
npm run build    # 构建游戏
npm run dev      # 构建+监听
npm run check    # 检查链接
```

直接打开 `dist/index.html` 即可游玩。
